### Category 2: System Fields (NOT from Form) - ~5 Fields

These fields are created/managed by GHL automatically:

1. `id` - GHL record ID (created by GHL)
2. `created_at` - Record creation timestamp (GHL system)
3. `updated_at` - Last update timestamp (GHL system)
4. `owner_id` - Record owner (GHL system)
5. `location_id` - Location context (GHL system)

**Note:** These are standard GHL custom object fields, not counted in the 109 custom fields.

---

### Category 3: Workflow/Approval Fields (NOT from Form) - ~5 Fields

These fields are used for internal workflow and approval processes:

1. `packager_approved` - Set by packager in GHL UI
2. `ba_approved` - Set by BA in GHL UI
3. `resubmit_for_testing` - Flag to resubmit for testing
4. `email_template_html` - Generated by Make.com, stored in GHL
5. `email_template_text` - Generated by Make.com, stored in GHL

**These are NOT populated from form submission.**

---

### Category 4: Project-Specific Fields (NOT for Single Property) - ~5 Fields

These fields are only used for Projects (Route 2), not Single Property (Route 1):

1. `lot_number` - Only for child records (projects)
2. `project_parent_id` - Only for child records (projects)
3. `project_identifier` - Only for projects (parent + children)
4. `is_parent_record` - Only for projects (parent = "Yes", child = "No")
5. `project_brief` - Project-specific content
6. `project_overview` - Project-specific content

**For Single Property:**
- `lot_number` = null or empty
- `project_parent_id` = null or empty
- `project_identifier` = null or empty
- `is_parent_record` = null or "No"
- `project_brief` = null or empty
- `project_overview` = null or empty

**These should still be included in Create JSON but set to null/empty for single property.**
